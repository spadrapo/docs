Configuration

Drapo is configured in Program.cs (or Startup.cs for .NET Core 3.1) using the AddDrapo service registration method and the UseDrapo middleware method. The DrapoMiddlewareOptions object passed to the lambda contains all configurable settings.

Minimal Setup

// Program.cs (.NET 6+)
builder.Services.AddDrapo();

app.UseDrapo();

Full Configuration Example

app.UseDrapo(options =>
{
    // Enable real-time pipes via SignalR
    options.Config.UsePipes = true;

    // Disable the built-in router if you manage routes manually
    options.Config.UseRouter = false;

    // Cache key strategies for views, components, and styles
    options.Config.CacheKeysView = "applicationbuild,url,view";
    options.Config.CacheKeysComponentView = "applicationbuild,url,view";
    options.Config.CacheKeysComponentStyle = "applicationbuild,url,theme";

    // Application build stamp used in cache keys
    options.Config.ApplicationBuild = typeof(Program).Assembly
        .GetName().Version?.ToString();

    // Themes (name, CSS suffix)
    options.Config.CreateTheme("", "");           // default theme
    options.Config.CreateTheme("Dark", "dark");   // dark theme

    // Responsive views
    options.Config.CreateView("Mobile", "mobile", "{{__browser.Width}} < 768");
    options.Config.CreateView("Web", "default");

    // Global error storage key
    options.Config.StorageErrors = "errors";

    // Validation CSS classes
    options.Config.ValidatorUncheckedClass = "ppValidationUnchecked";
    options.Config.ValidatorValidClass = "ppValidatorValid";
    options.Config.ValidatorInvalidClass = "ppValidatorInvalid";

    // Load components from a folder
    options.Config.LoadComponents(
        Path.Combine(env.WebRootPath, "components"),
        "~/components");

    // SPA routes
    options.Config.CreateRoute(
        "^/page/(?<id>\\w+)$",
        "UpdateSector(content,~/app/pages/{{id}}.html)");
    options.Config.CreateRoute(
        "^/$",
        "UpdateSector(content,~/app/pages/home.html)");

    // Polling for cache invalidation
    options.PollingEvent += async (domain, connectionId, key) =>
    {
        return await Task.FromResult(DateTime.UtcNow.Second / 5 + "");
    };
});

Key Configuration Properties

  • UsePipes – Enables the SignalR hub for real-time pipe updates.
  • UseRouter – When true, Drapo intercepts browser navigation and applies the configured routes.
  • CacheKeysView / CacheKeysComponentView / CacheKeysComponentStyle / CacheKeysComponentScript – Comma-separated tokens that define the cache key strategy for views and components. Common tokens include applicationbuild, url, view, and theme.
  • ApplicationBuild – A string that changes on each deployment, used as part of cache keys to bust caches automatically.
  • StorageErrors – The client-side storage key name where server error information is placed.
  • PipeHubName – The URL path suffix for the SignalR hub (default: DrapoPlumberHub).
  • ValidateWebSocketOrigin – Enables cross-origin WebSocket request validation (enabled by default).
  • AllowedWebSocketOrigins – An explicit allow-list of origins permitted to open WebSocket connections.

Debug Mode

When options.Debug = true, the Drapo middleware serves non-minified assets and outputs additional diagnostics. This is typically enabled only in the Development environment.

if (env.IsDevelopment())
    options.Debug = true;

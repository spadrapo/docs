Sectors

A sector is a named region of the page managed by Drapo. Sectors are the building blocks of Drapo's Single Page Application model: they allow parts of the page to be loaded, cleared, and replaced independently without a full page reload.

Defining a Sector

Mark any HTML element as a sector by adding the d-sector attribute with a unique name.

<div d-sector="content"></div>
<div d-sector="sidebar"></div>

Loading Content into a Sector

The UpdateSector function replaces the content of a sector with HTML loaded from a URL. It is typically called from route handlers or event handlers.

<!-- Load a page into the content sector -->
<button d-on-click="UpdateSector(content, ~/app/pages/home.html)">Home</button>

<!-- Clear a sector -->
<button d-on-click="ClearSector(sidebar)">Close Sidebar</button>

Nested Sectors

Sectors can be nested. A child HTML view loaded into a parent sector may itself define further sectors, enabling hierarchical layout composition. Child sectors are initialised after the parent sector's HTML is loaded and the Drapo runtime processes the new markup.

Sector Lifecycle

When a sector is updated (UpdateSector), Drapo:

  1. Fetches the HTML from the specified URL (using the browser cache or middleware cache if configured).
  2. Destroys any storage items and event bindings belonging to the current sector content.
  3. Replaces the sector's inner HTML with the newly loaded content.
  4. Processes Drapo attributes in the new content, initialising storage, bindings, and nested sectors.

Default Sector Navigation (Parent / Child)

Drapo provides d-parent / d-parentSector and d-child / d-childSector attributes to define default relationships between views. This allows a child page to declare which sector of the parent layout it should be loaded into, enabling convention-based navigation without explicit UpdateSector calls.

<!-- In a layout (parent) view -->
<div d-sector="main"></div>

<!-- In a child view -->
<div d-parent="layout.html" d-parentSector="main">
    <!-- child content -->
</div>

Sectors and SPA Routing

Drapo's routing system (configured via CreateRoute in startup) maps URL patterns to UpdateSector calls, so navigating to a route automatically loads the matching view into the correct sector. This is the primary mechanism for implementing navigation in a Drapo SPA without full page reloads.

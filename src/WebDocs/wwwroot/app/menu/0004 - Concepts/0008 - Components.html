Components

Drapo components are reusable, self-contained UI units composed of an HTML template, an optional CSS style sheet, and an optional TypeScript/JavaScript file. They are loaded and rendered by the Drapo middleware and can be used anywhere in a Drapo application using a custom HTML element that matches the component folder name.

Component Folder Structure

Each component lives in its own sub-folder under the configured components directory (typically wwwroot/components/). The folder name becomes the custom element tag name.

wwwroot/
  components/
    my-button/
      my-button.html    <!-- template -->
      my-button.less    <!-- styles (Less/CSS) -->
      my-button.ts      <!-- optional TypeScript -->

Registering Components

Tell the Drapo middleware where the components are located via LoadComponents in your startup configuration:

options.Config.LoadComponents(
    Path.Combine(env.WebRootPath, "components"),
    "~/components");

Using a Component

Once registered, use the component as a custom HTML element. The Drapo client will replace it with the rendered component template.

<my-button label="Save" d-on-click="PostData(user)"></my-button>

Component Inputs (Attributes)

HTML attributes placed on the component element are passed to the component template and are accessible inside it as storage properties or via expressions. This allows components to be configured at their usage site.

Component Packs

Multiple components can be bundled into a pack for efficient delivery. A pack is a single HTTP request that returns the CSS for all components in the bundle. Packs are configured with include/exclude glob paths:

options.Config.CreatePack("uicomponents")
    .AddIncludePath("~/components/*")
    .AddExcludePath("*.ts")
    .AddExcludePath("*.map");

Component Lifecycle

When Drapo renders a sector that contains component elements:

  1. The component HTML template is fetched from the middleware (or returned from cache).
  2. The template is inserted in place of the custom element.
  3. Drapo processes any d-* attributes in the template, enabling data binding and event handling within the component.
  4. The component's CSS (if any) is injected into the page if not already present.

Theming Components

Components participate in Drapo's theming system. You can provide theme-specific CSS files (e.g., my-button.dark.less) and Drapo will load the appropriate variant based on the active theme.

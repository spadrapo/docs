Routing

Drapo includes a client-side SPA routing system that maps URL patterns to Drapo function calls — typically UpdateSector calls that load different views. Routes are configured on the server and sent to the client as part of the Drapo initialisation payload.

Defining Routes

Routes are created in your startup configuration using CreateRoute(pattern, action). The pattern is a regular expression and the action is a Drapo function string.

// Match /page/{id} and load the corresponding view
options.Config.CreateRoute(
    "^/page/(?<id>\\w+)$",
    "ClearSector(content);UpdateSector(content,~/app/pages/{{id}}.html)");

// Default route — home page
options.Config.CreateRoute(
    "^/$",
    "UpdateSector(content,~/app/home.html)");

Named Capture Groups

Regular expression named capture groups (e.g., (?<id>\w+)) are automatically extracted and made available as expressions inside the action string. In the example above, {{id}} in the action is replaced by the captured value from the URL.

Applying a Route

Routes are applied programmatically using the ApplyRoute function. This updates the browser's address bar and executes the matching route action without a full page reload.

<a d-on-click="ApplyRoute(/page/about)">About</a>
<button d-on-click="ApplyRoute(/page/{{item.slug}})">Open</button>

Rejected Routes

Some URL paths should not be handled by the client-side router (e.g., API endpoints, static files). Add rejected route patterns to prevent Drapo from intercepting those URLs:

options.Config.CreateRouteRejected("^/api/");

Dynamic / Multi-Tenant Routes

For multi-tenant applications, you can inject tenant-specific routing logic using the RouteEvent delegate. The delegate receives the current domain and URL, and returns the action to execute. This allows per-tenant view resolution without duplicating route definitions.

options.Config.RouteEvent = async (domain, url) =>
{
    string tenant = ResolveTenant(domain);
    return $"UpdateSector(content,~/tenants/{tenant}/home.html)";
};

UseRouter Flag

Set options.Config.UseRouter = false to disable the Drapo client router entirely if you prefer to handle all navigation with explicit UpdateSector calls without changing the browser URL. This is useful for applications embedded inside other pages or iframes.

Getting Started with Drapo

Welcome to Drapo! This guide will help you get up and running with Drapo, the powerful frontend framework for .NET developers.

What is Drapo?

Drapo is a declarative frontend framework that brings reactive programming to .NET web applications. You can build complex single-page applications using only HTML attributes and .NET - no JavaScript required!

Prerequisites

Before you start, make sure you have:

  • .NET 8.0 SDK or later - Download here
  • Visual Studio, VS Code, or your preferred IDE
  • Basic knowledge of HTML and C#

Quick Installation

Setting up Drapo is simple. Follow these steps:

1. Create a new ASP.NET Core project

dotnet new web -n MyDrapoApp cd MyDrapoApp

2. Add the Drapo NuGet package

dotnet add package Drapo

3. Configure Drapo in your Startup.cs or Program.cs

public void ConfigureServices(IServiceCollection services) { services.AddDrapo(); }

For more detailed installation instructions, see the Installation section.

Your First Drapo Application

Let's create a simple interactive application to see Drapo in action. Create an index.html file in your wwwroot folder:

<!DOCTYPE html> <html> <head> <title>My First Drapo App</title> <script src="/drapo.js"></script> </head> <body> <div d-sector="main"> <!-- Data setup --> <div d-dataKey="user" d-dataType="object" d-dataProperty-name="John Doe" d-dataProperty-age="25"></div> <h1>Welcome to Drapo!</h1> <!-- Two-way data binding --> <p>Name: <input type="text" d-model="{{user.name}}" /></p> <p>Age: <input type="number" d-model="{{user.age}}" /></p> <!-- Display bound data --> <h2>Hello, {{user.name}}!</h2> <p d-if="{{user.age >= 18}}">You are an adult.</p> <p d-if="{{user.age < 18}}">You are a minor.</p> <!-- Event handling --> <button d-on-click="SetDataField(user, age, 30)">Set Age to 30</button> </div> </body> </html>

Run your application

dotnet run

Open your browser to http://localhost:5000 and you'll see your interactive Drapo application!

Key Concepts You Just Used

  • d-sector - Defines a data scope for your application
  • d-dataKey - Creates a data object named "user"
  • d-model - Creates two-way data binding with form inputs
  • {{expression}} - Displays data values in templates
  • d-if - Conditional rendering based on data
  • d-on-click - Event handling for user interactions

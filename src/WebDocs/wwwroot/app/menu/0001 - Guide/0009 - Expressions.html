Expressions

Drapo expressions are snippets of dynamic content embedded in HTML attribute values and text nodes using double curly braces: {{expression}}. The Drapo client evaluates these expressions in the context of the current sector's storage and updates the DOM whenever the referenced storage values change.

Basic Property Access

The simplest expression is a path to a property of a storage item.

{{user.name}}
{{order.total}}
{{settings.theme}}

Nested Properties

Expressions support arbitrarily deep property paths.

{{order.shipping.address.city}}

Array Item Properties (inside d-for)

Inside a d-for loop the iterator variable is accessible directly.

<li d-for="item in cart">{{item.name}} x {{item.quantity}}</li>

Arithmetic and Comparisons

Drapo supports basic arithmetic and comparison operators inside expressions used in conditional attributes such as d-if and d-class.

<!-- Show element only when count is greater than zero -->
<span d-if="{{cart.length}} > 0">{{cart.length}} items</span>

<!-- Apply a CSS class conditionally -->
<div d-class="{active:{{tab.selected}}}"></div>

Function Call Expressions

Drapo built-in functions can be invoked inside event attribute values (e.g., d-on-click). Multiple function calls can be chained with a semicolon.

<button d-on-click="SaveData(user);ClearDataField(form,error)">Save</button>

Special Variables

Drapo injects a set of special storage items that are always available:

  • __browser — Browser information such as __browser.Width and __browser.Height.
  • __location — Current URL information.
  • __sector — Sector metadata.
<!-- Show mobile layout for small screens -->
<div d-if="{{__browser.Width}} < 768">Mobile View</div>

Expressions in Attributes

Expressions can appear inside any HTML attribute value, not just text content. This allows dynamic URLs, class names, aria attributes, and more.

<a href="/user/{{user.id}}">Profile</a>
<img src="{{product.imageUrl}}" alt="{{product.name}}" />

Parameter Types

Functions and attributes accept different types of parameters. Understanding these types helps you use functions correctly:

  • text — A plain string value. Example: SaveData(user,"my label")
  • mustache — A data binding expression in {{double curly braces}}. Example: {{user.name}} or {{person.Address.City}}
  • boolean — A value of true or false
  • url — A valid URL path. Example: /api/users
  • conditional — An expression that evaluates to true or false. Example: {{user.isActive}} == true
  • expression — A collection of function calls separated by semicolons. Example: SaveData(user);Navigate(/home)
  • datakey — The name of a storage item declared with d-dataKey. Example: user, products
  • datafields — A dot-separated path within a storage item. Example: user.address.city
  • function — A function that evaluates and returns a value
  • validation — A validation rule or group of rules applied to a field

Escaping

If you need to output the literal text {{ without it being treated as an expression, encode the braces in HTML (&#123;&#123;) or consider using a server-side template to emit the final HTML.

Runtime Introspection

Runtime introspection lets external tooling inspect a running Drapo application — which sectors are mounted, which data keys are in storage, the browser/runtime state, and recent client diagnostics. It is intended for automated tooling such as test harnesses, LLM/agent drivers (Playwright/Selenium), and the built-in debugger that need to reason about the actually loaded page rather than the source markup.

Stable entry point

When the application loads, the Drapo instance is published on the global object. window.drapo is a string-keyed global and the public member names are preserved by the production minifier, so these accessors are safe to use against both drapo.js and drapo.min.js.

window.drapo                 // the DrapoApplication instance
window.drapo.Introspection   // query the running app (sectors, data keys, runtime state, snapshot)
window.drapo.Diagnostics     // client diagnostics buffer and sector images

Introspection API

The window.drapo.Introspection object exposes:

  • GetSectors() — names of the sectors currently mounted in the document (string[]).
  • GetDataKeys(sector?) — data keys in storage, optionally scoped to a sector (string[]).
  • IsDataKey(key, sector?) — whether key is a (non-hidden) data key (boolean).
  • GetRuntimeState() — { isLoaded, browser: { width, height } }.
  • GetSnapshot(diagnosticsCount?, diagnosticsLevelFilter?) — one call returning state + sectors + data keys by sector + diagnostics.

GetSnapshot

GetSnapshot() is the recommended entry point for tooling: a single round-trip returns everything needed to validate bindings against the live application. It composes the introspection primitives above with the diagnostics buffer.

const snapshot = window.drapo.Introspection.GetSnapshot();
// {
//   state: { isLoaded: boolean, browser: { width: number, height: number } },
//   sectors: string[],
//   dataKeysBySector: { [sector: string]: string[] },
//   diagnostics: DrapoDiagnosticEntry[]
// }

Diagnostics API

The window.drapo.Diagnostics object exposes the client diagnostics buffer and image capture:

  • GetErrorBuffer(count?, levelFilter?) — recent client diagnostics, errors and warnings (DrapoDiagnosticEntry[]).
  • GetSectorImage(sector?) — best-effort base64 PNG of a sector, or the document body when null (Promise<string>).

Example: reading the snapshot with Playwright

Drive a running app and assert against the live runtime state. The same pattern works with Selenium via execute_script.

import { test, expect } from '@playwright/test';

test('drapo runtime snapshot', async ({ page }) => {
    await page.goto('http://localhost:5000/');

    // Wait until Drapo has finished loading.
    await page.waitForFunction(() => {
        const drapo = window.drapo;
        return (drapo != null) && (drapo.Introspection.GetRuntimeState().isLoaded === true);
    });

    const snapshot = await page.evaluate(() => window.drapo.Introspection.GetSnapshot());

    expect(Array.isArray(snapshot.sectors)).toBeTruthy();
    // No client-side errors should have been captured while loading.
    expect(snapshot.diagnostics.filter(d => d.level === 'error')).toHaveLength(0);
});

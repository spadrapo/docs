Formatting

Drapo can format numbers, dates, and timespans directly in the template using the d-format attribute on any element that has a data binding. The formatter applies the specified pattern to the resolved value before rendering it to the DOM. Combine it with d-culture to render locale-aware output.

Numbers

Number format codes follow a letter + optional digit precision pattern:

  • N / N2 — Numeric with thousands separators. Default 2 decimal places.
  • D / D3 — Decimal integer, zero-padded to the specified number of digits.
  • P / P2 — Percentage (multiplies by 100, appends %).
  • T — Timespan: renders a millisecond value as 3d2h5m10s.
  • S — File size: renders a byte value as 1.2kb, 5mb, etc.
<div d-dataKey="price" d-dataType="value" d-dataValue="1234.5"></div>

<span d-model="{{price}}" d-format="N2"></span>   <!-- 1,234.50 -->
<span d-model="{{price}}" d-format="P2"></span>   <!-- 123,450.00 % -->

Dates

Date format codes map to common patterns. Short single-letter codes delegate to the active culture for their exact pattern:

  • d — Short date pattern (e.g. 06/03/1980)
  • D — Long date pattern (e.g. Tuesday, June 3, 1980)
  • t — Short time pattern (e.g. 11:57 AM)
  • T — Long time pattern (e.g. 11:57:24 AM)
  • g — General date/time, short time (e.g. 06/03/1980 11:57 AM)
  • G — General date/time, long time
  • r — RFC 1123 / HTTP date format

Custom date patterns use the following tokens:

  • YYYY / yy — Full / 2-digit year
  • MM / MMM / MMMM — Month: 2-digit / abbreviation / full name
  • DD / DDD / DDDD — Day: 2-digit / abbreviation / full weekday name
  • HH / hh — 24-hour / 12-hour, both zero-padded
  • mm / ss — Minutes / seconds
  • tt — AM/PM indicator
<div d-dataKey="event" d-dataType="object"
     d-dataProperty-date="1980-06-03T11:57:24Z"></div>

<span d-model="{{event.date}}" d-format="D"></span>
<!-- Tuesday, June 3, 1980 -->

<span d-model="{{event.date}}" d-format="DD/MM/YYYY HH:mm"></span>
<!-- 03/06/1980 11:57 -->

<span d-model="{{event.date}}" d-format="h:mm tt"></span>
<!-- 11:57 AM -->

Timespans

The T format code renders a raw millisecond integer as a human-readable duration:

<!-- 3661000 ms → 1h1m1s -->
<span d-model="{{task.durationMs}}" d-format="T"></span>

Locale-Aware Formatting with d-culture

Add d-culture to override the default culture for a specific element. This changes the decimal/thousands separator, month names, and AM/PM labels:

<!-- English: 1,234.50 -->
<span d-model="{{price}}" d-format="N2" d-culture="en"></span>

<!-- Portuguese: 1.234,50 -->
<span d-model="{{price}}" d-format="N2" d-culture="pt"></span>

<!-- Spanish long date -->
<span d-model="{{event.date}}" d-format="D" d-culture="es"></span>

Inline Formatting in d-for Loops

Formatting works naturally inside d-for loops, applied per-row:

<tr d-for="product in products">
    <td d-model="{{product.name}}"></td>
    <td d-model="{{product.price}}" d-format="N2"></td>
    <td d-model="{{product.releaseDate}}" d-format="d"></td>
</tr>

Formatting and d-attr

The d-attr attribute supports a d-attr-{name}-format companion to format values before placing them into an attribute:

<input type="text"
       d-attr-value="{{product.price}}"
       d-attr-value-format="N2" />

Server-Side Timezone Offset

Configure a server-side timezone offset so that UTC dates from the API are automatically shifted for the user. Set options.Config.Timezone in your startup and Drapo adds the offset before applying date formats.

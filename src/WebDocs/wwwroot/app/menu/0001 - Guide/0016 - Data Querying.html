Data Querying

Drapo supports a SQL-like client-side query engine that lets you combine, filter, sort, and project existing storage items into new derived datasets — entirely in the browser, without extra server calls. This is done using d-dataType="query" with a SQL expression in d-dataValue.

Basic Projection

Select specific fields from an existing array storage item. The query result is itself a new array that can be iterated with d-for:

<div d-dataKey="products" d-dataUrlGet="/api/products"></div>

<!-- Derived query: only Name and Price columns -->
<div d-dataKey="productNames"
     d-dataType="query"
     d-dataValue="SELECT Name, Price FROM products"></div>

<div d-for="p in productNames">
    <span>{{p.Name}}</span> — <span>{{p.Price}}</span>
</div>

Joins

Query supports INNER JOIN, LEFT JOIN, and OUTER JOIN across multiple storage items, using table aliases and an ON condition:

<!-- INNER JOIN: only rows that match in both arrays -->
<div d-dataKey="ordersJoined"
     d-dataType="query"
     d-dataValue="SELECT O.Id, O.Qty, P.Name
                  FROM orders AS O
                  INNER JOIN products AS P ON O.ProductId = P.Id"></div>

<!-- LEFT JOIN: all rows from the left + matching rows from the right -->
<div d-dataKey="personsSelected"
     d-dataType="query"
     d-dataValue="SELECT S AS Selected, P.Code, P.Name
                  FROM persons AS P
                  LEFT JOIN selections AS S ON P.Code = S"></div>

<!-- OUTER JOIN: all rows from both sides, nulls where unmatched -->
<div d-dataKey="fullList"
     d-dataType="query"
     d-dataValue="SELECT COALESCE(O1, O2) AS Code, O1 AS Code1, O2 AS Code2
                  FROM list1 AS O1
                  OUTER JOIN list2 AS O2 ON O1 = O2"></div>

COALESCE(a, b) returns the first non-null value — useful with OUTER JOIN to produce a merged key column.

WHERE Clause

Filter rows using standard comparison operators, IS NULL / IS NOT NULL, LIKE, and mustache bindings for dynamic values:

<!-- Static filter -->
<div d-dataKey="activeUsers" d-dataType="query"
     d-dataValue="SELECT Code, Name FROM persons WHERE Code = 2"></div>

<!-- Dynamic filter using mustache -->
<div d-dataKey="filteredUsers" d-dataType="query"
     d-dataValue="SELECT Code, Name FROM persons WHERE Code = {{selectedCode}}"></div>

<!-- Null checks -->
<div d-dataKey="noAge" d-dataType="query"
     d-dataValue="SELECT Name FROM persons WHERE Age IS NULL"></div>

<!-- Pattern match -->
<div d-dataKey="startsWithL" d-dataType="query"
     d-dataValue="SELECT Code FROM items WHERE Code LIKE 'L%'"></div>

Because the query re-evaluates whenever a referenced mustache value changes, this pattern is a natural way to build reactive filtered lists.

ORDER BY

Sort the result set by one or more columns. Both ASC and DESC are supported, and you can dynamically bind the sort column/direction with mustaches:

<!-- Static sort -->
<div d-dataKey="sortedPersons" d-dataType="query"
     d-dataValue="SELECT Code, Name FROM persons ORDER BY Name ASC"></div>

<!-- Dynamic sort -->
<div d-dataKey="sort" d-dataType="object"
     d-dataProperty-column="Name"
     d-dataProperty-direction="ASC"></div>
<div d-dataKey="dynamicSort" d-dataType="query"
     d-dataValue="SELECT Code, Name FROM persons ORDER BY {{sort.column}} {{sort.direction}}"></div>

DISTINCT

Remove duplicate rows from the result. Works with single or multiple columns:

<div d-dataKey="uniqueGenders" d-dataType="query"
     d-dataValue="SELECT DISTINCT Gender FROM persons"></div>

<div d-dataKey="uniqueProducts" d-dataType="query"
     d-dataValue="SELECT DISTINCT Product, Quantity FROM orders"></div>

Aggregation

Query supports the aggregate functions COUNT, MAX, MIN, SUM, and AVG. An aggregate query computes a single aggregate over one column: the result is a single object (not an array), accessed by its alias as {{key.Alias}}:

<div d-dataKey="orders" d-dataType="array"
     d-dataValue='[{"Id":1,"Amount":10},{"Id":2,"Amount":20},{"Id":3,"Amount":30}]'></div>

<!-- aggregate query → single object result -->
<div d-dataKey="orderTotal" d-dataType="query"
     d-dataValue="SELECT Sum(Amount) AS Total FROM orders"></div>

<!-- Access as object property, not array item -->
<span>Total: {{orderTotal.Total}}</span>

Each query evaluates one aggregate function. To show several totals from the same source, use a separate query storage item per aggregate:

<div d-dataKey="orderTotal"   d-dataType="query" d-dataValue="SELECT Sum(Amount) AS Total   FROM orders"></div>
<div d-dataKey="orderAverage" d-dataType="query" d-dataValue="SELECT Avg(Amount) AS Average FROM orders"></div>
<div d-dataKey="orderItems"   d-dataType="query" d-dataValue="SELECT Count(Id)   AS Items   FROM orders"></div>

SUM(column) returns the numeric total of the column's values; null and non-numeric values are skipped, and it returns null when there are no numeric values. AVG(column) returns the sum divided by the count of the numeric values (skipping the same way), or null when there are none. COUNT, MAX, and MIN behave as their SQL equivalents.

Aggregate queries are reactive (see Reactivity below): when the source storage changes, the aggregate re-evaluates automatically — convenient for live totals.

Engine version: SUM and AVG require the Drapo engine shipped in 2026.6.30.2 or later. On earlier builds Sum(...)/Avg(...) parse and validate but return an empty result (the engine falls back to a normal projection) — a missing total is the tell-tale sign the engine needs upgrading. COUNT, MAX, and MIN are available on earlier builds.

Mustache Sources

The FROM clause can reference nested data paths using mustache expressions, letting you query a sub-array of an object:

<div d-dataKey="catalog" d-dataUrlGet="/api/catalog"></div>

<!-- Query from a nested array within catalog -->
<div d-dataKey="featuredItems" d-dataType="query"
     d-dataValue="SELECT Name, Price FROM {{catalog.featured}}"></div>

Primitive Array Output

By default a query always produces an array of objects. To get a primitive array (a flat list of values), add the d-dataQueryArray attribute and specify which column to extract:

<div d-dataKey="codes" d-dataType="query"
     d-dataQueryArray="Code1"
     d-dataValue="SELECT O1 AS Code1 FROM list1 AS O1"></div>

<!-- Iterate as primitive: {{item}} not {{item.Code1}} -->
<div d-for="code in codes">
    <span>{{code}}</span>
</div>

Reactivity

Query storage items are reactive. Whenever the source storage items change (e.g., after a ReloadData or a d-model update), all dependent query storage items automatically re-evaluate and re-render their bound elements.

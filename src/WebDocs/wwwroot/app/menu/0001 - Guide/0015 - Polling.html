Polling

Polling lets a storage item automatically refresh its data at a regular interval by calling a server-side endpoint. When the server reports a change (by returning a different hash), Drapo executes a configured expression — typically a ReloadData call — to re-fetch and re-render the data.

How It Works

  1. A "trigger" storage item is declared with d-dataPollingKey and d-dataPollingTimespan.
  2. Drapo sends the polling key to your server at the configured interval.
  3. Your server returns a hash string representing the current state of the data.
  4. If the hash differs from the last one, Drapo executes the storage item's d-dataValue expression (e.g. ReloadData(myKey)).

Server-Side: Register the Polling Event

Add a PollingEvent handler in your startup configuration. It receives the domain, connection ID, and the polling key, and must return a hash string. When the hash changes, Drapo triggers a reload:

options.PollingEvent += async (domain, connectionId, key) =>
{
    // Return a hash that changes when the data changes.
    // Drapo will only reload when this value differs from the previous call.
    string hash = await GetDataHashForKey(domain, key);
    return hash;
};

A simple example using a time-bucket as the hash (changes every 5 seconds):

options.PollingEvent += async (domain, connectionId, key) =>
{
    string hash = Math.Ceiling((decimal)(DateTime.Now.Second / 5)).ToString();
    return await Task.FromResult(hash);
};

Client-Side: Configure the Storage Item

Create a dedicated "trigger" storage item of type function with load type startup. Set its polling key and timespan, and provide the reload expression as its value:

<!-- The actual data storage item -->
<div d-dataKey="liveData" d-dataType="value" d-dataUrlGet="/api/live"></div>

<!-- The polling trigger — runs ReloadData(liveData) when the hash changes -->
<div d-dataKey="liveDataPoller"
     d-dataType="function"
     d-dataLoadType="startup"
     d-dataValue="ReloadData(liveData)"
     d-dataPollingKey="liveData"
     d-dataPollingTimespan="5000"></div>

<span d-model="{{liveData}}"></span>

Key Attributes

  • d-dataPollingKey — A unique string key identifying this polling item. Sent to the server on each poll call.
  • d-dataPollingTimespan — Interval in milliseconds between polls. Default: 10000 (10 seconds).
  • d-dataValue — The Drapo function expression executed when a change is detected. Typically ReloadData(dataKey).
  • d-dataLoadType — Set to startup so the poller activates immediately when the sector is rendered.

Polling Multiple Items

Each storage item you want to poll needs its own trigger item with a unique d-dataPollingKey:

<div d-dataKey="orders" d-dataType="array" d-dataUrlGet="/api/orders"></div>
<div d-dataKey="ordersPoller" d-dataType="function" d-dataLoadType="startup"
     d-dataValue="ReloadData(orders)"
     d-dataPollingKey="orders" d-dataPollingTimespan="3000"></div>

<div d-dataKey="alerts" d-dataType="array" d-dataUrlGet="/api/alerts"></div>
<div d-dataKey="alertsPoller" d-dataType="function" d-dataLoadType="startup"
     d-dataValue="ReloadData(alerts)"
     d-dataPollingKey="alerts" d-dataPollingTimespan="10000"></div>

Polling vs. Pipes

Polling is a simple pull-based mechanism: the client asks "has anything changed?" at a regular interval. For push-based real-time communication (the server notifies clients of changes), use Pipes (SignalR-backed). Polling is simpler to set up; Pipes are more efficient for high-frequency updates.

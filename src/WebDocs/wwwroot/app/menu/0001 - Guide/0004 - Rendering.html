Rendering

Drapo's rendering system lets you project dynamic data directly into the DOM using declarative attributes — no JavaScript required. The core tool is d-for for iterating collections, combined with d-if for conditional rendering.

Basic Array Rendering with d-for

Place d-for on any element to repeat it for every item in an array storage key. The syntax is {alias} in {dataKey}:

<div d-dataKey="products" d-dataUrlGet="/api/products"></div>

<ul>
    <li d-for="product in products">
        {{product.name}} — {{product.price}}
    </li>
</ul>

Drapo automatically re-renders the list whenever the products storage changes.

Iterating Objects

When the iterator target is an object (not an array), Drapo converts it into a list of Key / Value pairs:

<div d-dataKey="user" d-dataType="object"
 d-dataProperty-name="Jane" d-dataProperty-role="Admin"></div>

<ul>
    <li d-for="prop in user">{{prop.Key}}: {{prop.Value}}</li>
</ul>

Range Slicing

Drapo supports range expressions inside d-for to render a subset of an array. Use the ^ operator to index from the end:

<!-- Reverse the list -->
<li d-for="item in items[^0..0]">{{item.name}}</li>

<!-- Skip first and last -->
<li d-for="item in items[1..^1]">{{item.name}}</li>

<!-- First 5 items -->
<li d-for="item in items[0..5]">{{item.name}}</li>

Conditional Items with d-if

Add d-if inside a d-for to filter which items are rendered. The element is completely removed from the DOM when the condition is false:

<li d-for="product in products" d-if="{{product.inStock}}">
    {{product.name}}
</li>

Conditionally Activating the Loop with d-for-if

Use d-for-if to enable or disable the entire loop based on an expression. When false, nothing is rendered at all:

<li d-for="item in cart" d-for-if="{{cart.length}} > 0">
    {{item.name}}
</li>

Controlling Re-render with d-for-render

d-for-render controls the re-render strategy. By default Drapo re-renders the whole list on any change. Set it to item to patch individual rows, which is more efficient for large lists:

<tr d-for="row in tableData" d-for-render="item">
    <td>{{row.id}}</td>
    <td>{{row.label}}</td>
</tr>

Nested Loops

Nest d-for attributes to render multi-level structures. Each loop has its own alias and they do not conflict:

<div d-for="category in categories">
    <h3>{{category.name}}</h3>
    <ul>
        <li d-for="product in category.products">
            {{product.name}}
        </li>
    </ul>
</div>

Rendering with Server-Loaded Data

Combine d-dataUrlGet with d-for to load and render server data in one step. The list renders as soon as the request completes:

<div d-dataKey="orders" d-dataUrlGet="/api/orders?status=pending"></div>

<table>
    <thead><tr><th>Order</th><th>Total</th><th>Date</th></tr></thead>
    <tbody>
        <tr d-for="order in orders">
            <td>{{order.id}}</td>
            <td>{{order.total}}</td>
            <td>{{order.date}}</td>
        </tr>
    </tbody>
</table>

Editing Items In-Place

Use d-model inside a d-for to make each row directly editable. Changes are tracked in storage and can be posted back to the server:

<tr d-for="item in items">
    <td><input d-model="{{item.name}}" /></td>
    <td><input d-model="{{item.price}}" type="number" /></td>
    <td>
        <button d-on-click="RemoveDataItem(items,{{item}})">Delete</button>
    </td>
</tr>
<button d-on-click="AddDataItem(items,{})">Add Row</button>
<button d-on-click="PostData(items,/api/items)">Save All</button>

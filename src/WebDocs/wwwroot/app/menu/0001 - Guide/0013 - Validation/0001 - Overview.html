Validation

Drapo provides a declarative, attribute-driven validation system. Validators are declared once as hidden elements and then referenced by interactive elements to block actions when invalid, and by display elements to show error messages — all without any custom JavaScript.

Defining a Validator

A validator is a <div> that carries the d-validation-id, d-validation-type, and one or more d-validation-value attributes:

<div d-validation-id="emailRequired"
     d-validation-type="conditional"
     d-validation-value="{{form.email}}"></div>

Validator Attributes

  • d-validation-id — unique name for the validator, referenced by actions and display elements
  • d-validation-type — the validation algorithm: conditional, regex, compare, or outside
  • d-validation-value — the primary value to test (usually a mustache expression)
  • d-validation-expression — regular expression pattern used with the regex type
  • d-validation-valuetocompare — the value to compare against, used with the compare type

Triggering Validation on User Actions

Add d-validation-on-click to any button or clickable element. Drapo runs all listed validators before executing the bound function. If any validator fails, the function is not called.

<button d-on-click="PostData(form,/api/register)"
        d-validation-on-click="emailRequired">Register</button>

Chain multiple validators with a semicolon — all must pass for the action to proceed:

d-validation-on-click="emailRequired;emailFormat;passwordMatch"

Displaying Validation Errors

Wrap error content in an element with d-validation pointing at the validator id. The element is shown only when the validator has failed and hidden when it passes:

<div d-validation="emailRequired">
    <span>Email address is required.</span>
</div>

Validation Types

  • conditional — passes when the expression evaluates to a truthy value (true, non-empty string, non-zero number). See the conditional sub-page.
  • regex — passes when d-validation-value matches the pattern in d-validation-expression. See the regex sub-page.
  • compare — passes when d-validation-value equals d-validation-valuetocompare. See the compare sub-page.
  • outside — delegates validation to an external function or server call, letting you implement any custom logic. See the outside sub-page.

Complete Example: Registration Form

<!-- Data -->
<div d-dataKey="form" d-dataType="object"
     d-dataProperty-email=""
     d-dataProperty-password=""
     d-dataProperty-confirm=""></div>

<!-- Validators -->
<div d-validation-id="emailRequired"
     d-validation-type="conditional"
     d-validation-value="{{form.email}}"></div>

<div d-validation-id="emailFormat"
     d-validation-type="regex"
     d-validation-value="{{form.email}}"
     d-validation-expression="^[\w.-]+@[\w.-]+\.\w{2,}$"></div>

<div d-validation-id="passwordMatch"
     d-validation-type="compare"
     d-validation-value="{{form.confirm}}"
     d-validation-valuetocompare="{{form.password}}"></div>

<!-- Form fields -->
<input type="email"    d-model="{{form.email}}"    placeholder="Email" />
<input type="password" d-model="{{form.password}}" placeholder="Password" />
<input type="password" d-model="{{form.confirm}}"  placeholder="Confirm password" />

<!-- Submit button — blocked until all three validators pass -->
<button d-on-click="PostData(form,/api/register)"
        d-validation-on-click="emailRequired;emailFormat;passwordMatch">
    Register
</button>

<!-- Error messages -->
<div d-validation="emailRequired"><span>Email is required.</span></div>
<div d-validation="emailFormat"><span>Enter a valid email address.</span></div>
<div d-validation="passwordMatch"><span>Passwords do not match.</span></div>

Best Practices

  • Place all validator <div> elements near the top of the form, after your data declarations.
  • Use meaningful d-validation-id names that describe the rule, not the field (e.g. emailFormat rather than field1).
  • Keep error message elements adjacent to their corresponding fields for the best user experience.
  • For complex server-side rules, use the outside type to call an API endpoint and reflect the result back into a storage field used by a conditional validator.

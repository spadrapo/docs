conditional

The conditional validation type can be used to validate a conditional expression.

Truthiness Rules

A conditional validator passes (allows the action) when the d-validation-value expression resolves to a truthy value, and fails (blocks the action) when it resolves to a falsy value.

ValueResult
"true" (string)✅ passes
"false" (string)❌ fails
"" (empty string)❌ fails
{} (empty object)✅ passes — objects are always truthy
true (boolean)✅ passes
false (boolean)❌ fails
non-zero number✅ passes
0 (number)❌ fails
null / undefined❌ fails

Important: An empty object {} created by d-dataType="object" is truthy. Use d-dataType="value" with an empty d-dataValue="" when you need a falsy initial state for a flag used in conditional validation. See the d-dataType attribute page for details.

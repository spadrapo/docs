This page collects practical patterns for common Drapo scenarios involving validation, data binding, and conditional rendering.

Stable Labels During User Input (New vs Edit Mode)

A label that switches between "New item" and "Edit item" based on whether a Name field is empty is naturally reactive: as soon as the user types a character, the label switches — even though they are still creating a new record.

Wrong approach — label switches while the user types:

<label d-model="{{text.NewItem}}" d-if="{{data.Name.length}}=0"></label> <label d-model="{{text.EditItem}}" d-if="{{data.Name.length}}>0"></label>

Correct approach — use a stable flag set once at startup:

Introduce a d-dataType="value" flag that is set once when the panel opens and never changes while the user types:

<!-- Flag starts empty (falsy) --> <div d-dataKey="panelIsNew" d-dataType="value" d-dataValue=""></div> <!-- Startup: set to true only when creating a new record (Name is empty) --> <div d-dataKey="panelStartup" d-dataType="function" d-dataLoadType="startup" d-dataValue="IF({{data.Name.length}}=0,UpdateData(panelIsNew,true))"></div> <!-- Labels use the stable flag, not the reactive Name length --> <label d-model="{{text.NewItem}}" d-if="{{panelIsNew}}"></label> <label d-model="{{text.EditItem}}" d-if="!{{panelIsNew}}"></label>

The flag is evaluated once when the panel opens and remains constant while the user edits the form.

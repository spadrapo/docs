d-dataCacheKeys

The d-dataCacheKeys attribute is used to specify one or more cache keys for a data source. This attribute allows Drapo to cache different requests based on specific keys, enabling partial caching based on dynamic values.

When a data retrieval operation is performed (typically via d-dataUrlGet), Drapo uses the values of these keys to build a unique cache entry. This mechanism enables different requests (for example, details of different users or products) to be cached separately, optimizing performance by avoiding unnecessary server requests when valid cached data is available.

The d-dataCacheKeys attribute should be used when your data source URL is dynamic—especially when it depends on parameters like user IDs, filter criteria, or other values that determine unique cache entries.


Sample 1 - Cache by Single Key

In this example, user profiles are cached separately by userId. Each distinct user's profile is stored and served from the cache when requested again.



Name: {{s1-userProfile.Name}}
Email: {{s1-userProfile.Email}}

Sample 2 - Cache by Multiple Keys

You can specify multiple cache keys separated by commas. In this example, products are cached based on both category and page number, ensuring that different combinations are cached separately.



  • {{product.Name}} - ${{product.Price}}

Related Attributes

  • d-dataCache - Controls whether the storage item is maintained in cache or used and discarded
  • d-dataUrlGet - Specifies the URL for GET operations to retrieve data
  • d-dataKey - Defines the unique identifier for a storage item

d-for-render

You can use d-for-render to control how the behavior of d-for.

You have a few options that can be used in this attribute:

  • viewport. This will inform the d-for that he need to create a viewport to render the data. So the render startup will be much more faster.
  • dom. When rendering the difference. The difference of nodes in the template will be compared to check if there is a change in the template.
  • html. When rendering the difference. The html of the template will be used to compare if there is any change in the template.
  • hash. When rendering the difference. A hash of all mustaches of the template will be used to check if there is a change in the template.

Examples of use:

d-for-render='viewport'

d-for-render='viewport,html'

d-for-render='viewport,hash'

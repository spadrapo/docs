d-dataType

You can use d-dataType to identify the type of the storage item.

The possible values are:

  • [value](value)
  • [object](object)
  • [array](array)
  • [function](function)
  • [querystring](querystring)
  • [file](file)
  • [mapping](mapping)
  • [parent](parent)

In this case, the storage item has the name obj1 and his type is [object](object)

Choosing Between value and object for Boolean Flags

When you need a simple boolean or scalar flag to drive a d-if expression, always use d-dataType="value" — not d-dataType="object".

An empty d-dataType="object" element is initialised as {}, which is an empty object. Empty objects are truthy in Drapo, so d-if="{{myFlag}}" will show the element even when you intend it to be hidden.

<!-- WRONG: empty object is truthy — d-if="{{myFlag}}" will show the element --> <div d-dataKey="myFlag" d-dataType="object" d-dataValue=""></div> <!-- CORRECT: empty string is falsy — d-if="{{myFlag}}" will hide the element --> <div d-dataKey="myFlag" d-dataType="value" d-dataValue=""></div>

Set the flag to truthy using a startup function:

<div d-dataKey="myFlagStartup" d-dataType="function" d-dataLoadType="startup" d-dataValue="IF({{someCondition}},UpdateData(myFlag,true))"></div>

Clear it back to falsy with:

UpdateData(myFlag,)

This sets myFlag to an empty string, which is falsy in d-if.

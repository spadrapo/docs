The LoadPack function loads a predefined pack of files from the server. A pack is a collection of files (HTML, CSS, JavaScript, etc.) that are bundled together on the server and delivered as a single compressed file to improve performance and reduce the number of HTTP requests.

When LoadPack is called with a pack name, it:

  • Retrieves the bundled pack - Downloads all files defined in the pack as a single compressed file from the server
  • Unpacks the content - Automatically extracts and places each file in its correct location in the frontend
  • Caches the files - Makes the unpacked files immediately available for use by other Drapo functions like UpdateSector

Packs are defined in the Drapo configuration and support:

  • Wildcard patterns - Include multiple files using patterns like *.html or components/**/*.js
  • Exclusion patterns - Exclude specific files or patterns from the pack
  • Multiple paths - Include files from different directories in a single pack
  • Compression - Files are automatically compressed (Brotli/Gzip/Deflate) for optimal performance

Dynamic Packs

Packs can also be configured as dynamic, allowing their content to be generated at runtime through a delegate handler. Dynamic packs provide:

  • Runtime content generation - Pack content is generated on-demand when requested, allowing for user-specific or context-dependent content
  • Delegate handler - A configurable HandlerPackDynamic delegate in DrapoConfig that receives the pack request and returns dynamically generated content
  • HTTP caching support - Dynamic packs support ETags for proper HTTP caching, reducing unnecessary data transfer
  • Flexible content structure - Content can be provided as a structured list of files or as pre-serialized JSON

Dynamic packs are created by setting the IsDynamic flag when defining the pack in the configuration, and configuring a delegate handler that will be called to generate the pack content each time it's requested.

This function is particularly useful for loading component libraries, themes, or any collection of related files that should be loaded together as a unit, whether predefined or generated dynamically.
